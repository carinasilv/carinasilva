Goal-Setting App

UX Research · Mobile App Design · Spring 2026

An end-to-end concept for a mobile app that helps users define long-term goals, set realistic timelines (including expected lulls in progress), and choose between shared or self-accountability. The project covered task persona research, task flow design, paper prototyping, three rounds of moderated user testing, style guide implementation, grid system applications and iterative redesign based on real tester feedback.

Task Flows

Three core flows: Creating a goal, Setting a timeline, and Sharing accountability. The full task flow document shows every screen and decision branch.

View task flows (PDF)

User Testing

Three moderated paper-prototype sessions with testers Willa (31), Maggie (23), and Thomas (21). Each session surfaced distinct friction points — from CTA placement and missing date pickers, to the cognitive load of the accountability screen with three near-equal options.

Key findings drove specific redesigns: removing redundant edit CTAs, adding pre-filled prompt prefixes ("I want to…", "I need to…") to reduce typing friction, introducing a proper date picker module, and rebalancing visual hierarchy on the accountability screen.

Read full user testing report (PDF)

Final Designs

The visual design applies a dark theme with warm orange accent color, refined CTA hierarchy, and the information architecture refined through testing.

View final designs (PDF)

Outcome

The iterative testing process surfaced multiple high-value design changes that wouldn't have been visible from the original task flows alone — particularly around reducing cognitive load and creating clearer visual hierarchy between primary and secondary actions. The project demonstrates an end-to-end UX process from concept through validated design.
